# Phase 8: Finalize — Push, PR, and JIRA Update

## Prerequisites

- All implementations complete and reviewed
- All tests passing
- Plan document is up to date

---

## Step 1: Verify and Push All Changes

1. **Check for uncommitted changes:**
   ```bash
   git status --short
   ```
   If uncommitted changes exist, stage specific files and commit.

2. **Push to remote:**
   ```bash
   git push -u origin feat/{ticket-id-lowercase}
   ```

3. **Verify tests pass one final time** using the project's test command.

4. **Verify lint passes** using the project's lint command.

If any step fails, resolve before proceeding.

---

## Step 2: Create PR

Read the PR body template from `${CLAUDE_PLUGIN_ROOT}/templates/pr-body.md` and create the PR:

```bash
gh pr create \
  --title "{commit-type}: {ticket-id-lowercase} {brief description}" \
  --body "$(cat <<'EOF'
## Summary

- {bullet point 1: what changed}
- {bullet point 2: what changed}
- JIRA: [{ticket-id}]({jira-url})

## Changes

{list of key files modified and why}

## Test Results

- Tests: {pass count} passing
- Lint: clean

## Plan

Plan document: `{plan-dir}/{TICKET-ID}/plan.md`

---
Generated by Kai Agent
EOF
)"
```

**Record the PR URL.**

---

## Step 3: Update JIRA

Using JIRA MCP tools:

1. **Add completion comment** using `${CLAUDE_PLUGIN_ROOT}/templates/jira-comment-complete.md` template:
   - PR URL, summary of changes, test results

2. **Optionally transition status** — ask before transitioning: "Should I move the ticket to 'In Review'?"

---

## Step 4: Update Plan Document

Read and update `{plan-dir}/{TICKET-ID}/plan.md`:

1. Mark ALL tasks as `"COMPLETED"`
2. Add a `## Results` section:

```markdown
## Results

**Completed:** {date}

### Pull Request
| PR | Status |
|-----|--------|
| [{PR-title}]({PR-url}) | Open |

### Summary
- {total files} files changed
- {total tests} tests passing
- Code review passed
- JIRA updated: {ticket-id}
```
